# Versioning Strategy

本文件定义 skill 自身的版本演进策略。

## 当前版本

当前版本：`project-create-rocky-0-1-0`

含义：

```text
rocky      个人/系列代号
0         大版本，规则体系仍在探索
1         小版本，已具备可用流程
0         修订号，文档和模板级别调整
```

## 版本号规则

- `0.1.x`: 修正文档、补模板、减少重复，不改变核心文件名。
- `0.2.0`: 可以增加新流程、新模板、新技术栈支持，但要兼容 0.1.x 项目。
- `1.0.0`: 流程稳定，文件结构和角色规则进入长期兼容。

## 升级规则

升级旧项目时：

1. 先读旧项目的 `README.md`、`HANDOFF.md`、`PROJECT_STRUCTURE.md`。
2. 只追加缺失的小节或文件。
3. 不覆盖已有 PRD、任务、交接记录。
4. 如果新旧规则冲突，先写入 `HANDOFF.md` 的风险和建议，再让用户确认。
