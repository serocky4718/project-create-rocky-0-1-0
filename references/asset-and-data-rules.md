# Asset And Data 资源与数据规则

用于管理图片、音频、视频、生成文件、私密数据和正式资源。

## AI 生成资源流程

AI 生成内容不要直接进入正式运行目录。

推荐流程：

```text
output/ 或 tmp/
-> 预览
-> 用户确认
-> public/assets/<kind>/ 或正式内容目录
-> 更新引用
-> 验证
```

## 目录建议

```text
output/images/      候选图片
output/audio/       候选音频
output/video/       候选视频
docs/design/        设计说明和参考
public/assets/      正式运行资源
storage/            私密运行数据
```

## 禁止事项

- 正式代码不要引用 `output/` 下的临时文件。
- 私密数据不要放到 `public/`。
- 不要提交真实账号、密钥、session、用户存档。
- 不要把设计草稿当作正式资源，除非用户确认。

## 新手解释模板

处理资源时说明：

```text
这个文件现在是候选资源还是正式资源
我准备把它放到哪里
为什么放在那里
页面或代码会如何引用它
成功标志是什么
```
